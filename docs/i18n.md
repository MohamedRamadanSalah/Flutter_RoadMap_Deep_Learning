# Internationalization (i18n)

## Overview

Slang provides type-safe, compile-time checked translations with autocompletion.
Raw string lookups are prohibited.

## Configuration

Slang config is in `slang.yaml` at project root. English is the base locale.

## Adding Translations

Edit `assets/i18n/en.i18n.yaml`:

```yaml
app:
  name: Hayyacom
auth:
  login: Login
  logout: Logout
  email_placeholder: Enter your email
errors:
  network: No internet connection
  unknown: Something went wrong
```

Regenerate:

```sh
dart run slang
```

## Using in Code

```dart
import 'package:hayyacom/core/i18n/strings.g.dart';

// Method A: Simple (no rebuild on locale change)
Text(t.auth.login)

// Method B: Via context (rebuilds on locale change)
Text(context.t.auth.login)
```

## Adding a New Locale

1. Create `assets/i18n/<locale>.i18n.yaml` (e.g., `ar.i18n.yaml`).
2. Add all keys from the English file.
3. Run `dart run slang`.

## Rules

- **DO** use `t.key.subkey` for all user-facing strings.
- **DO NOT** use hardcoded strings in UI code.
- **DO NOT** use raw string lookups (`t['key']`) except for edge cases.
